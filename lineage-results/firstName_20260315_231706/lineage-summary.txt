firstName is a person name attribute stored in two database tables in the spring-petclinic application: owners.first_name (VARCHAR 30) and vets.first_name (VARCHAR 30), both defined across MySQL, PostgreSQL, and H2 schemas. The field is defined once in the shared base class Person.java (@MappedSuperclass) and inherited by both Owner.java and Vet.java JPA entities, making it a single canonical source of truth used across two domain objects. For the Owner domain, firstName flows inbound from HTML form submissions at POST /owners/new and POST /owners/{id}/edit — Spring MVC binds the form field to the Owner object via @ModelAttribute, which is then persisted to the database via OwnerRepository.save(); it flows outbound via GET /owners/{id} (ownerDetails.html) and GET /owners (ownersList.html) Thymeleaf views. For the Vet domain, firstName is read-only (populated only by seed data) and exposed outbound via a REST JSON endpoint at GET /vets (Jackson serialization) and a Thymeleaf view at GET /vets.html. The only transformation in the entire chain is a JPA naming convention rename: the database column first_name (snake_case) is automatically mapped to the Java field firstName (camelCase) on read, and reversed to first_name on write — no custom mappers, stored procedures, type converters, or format changes exist anywhere in the lineage.
